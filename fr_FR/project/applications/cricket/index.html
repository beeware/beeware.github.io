Cricket

L'interface graphique de l'exécution des tests qui détecte rapidement les défaillances

Cricket est un outil graphique qui aide les projets avec de grandes suites de tests à identifier les défaillances sans attendre que tous les tests soient terminés.

Les programmes de test unittest normaux envoient toutes les données de sortie à la console et fournissent très peu de détails pendant l'exécution de la suite de tests. très peu de détails pendant l'exécution de la suite. En conséquence :

  • Vous ne pouvez pas commencer à examiner les échecs tant que la suite de tests n'a pas été exécutée. terminé son exécution,
  • Ce n'est pas un format très accessible pour identifier des modèles dans les des tests,
  • Il peut être difficile (ou encombrant) de réexécuter les tests qui ont échoué.

Comment aider

Doit être porté sur Toga

Pourquoi est-ce nommé "Cricket"?

Le [Test Cricket] (https://en.wikipedia.org/wiki/Test_cricket) est la version la plus prestigieuse du jeu de cricket. version la plus prestigieuse du cricket. Les parties durent jusqu'à 5 jours jours... tout comme l'exécution de certaines suites de tests. L'approche habituelle pour pour rendre le cricket regardable est une généreuse dose de bière ; en programmation, en programmation, les limites du [Balmer Peak] (https://xkcd.com/323/) entrent en vigueur, donc autre chose est nécessaire...
