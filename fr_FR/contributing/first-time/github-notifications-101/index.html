GitHub Notifications 101

Vous utilisez maintenant GitHub et vous en êtes informé. Et maintenant ?

Une fois que vous aurez commencé à contribuer à BeeWare, GitHub commencera à vous à vous notifier des choses.

C'est une bonne chose ! Mais cela peut devenir insurmontable. Voici donc quelques conseils pour vous aider à lutter contre le déluge.

  • Si vous préférez recevoir uniquement des notifications par courrier électronique, désactivez les notifications "Web". les notifications "Web".
  • Si vous préférez ne pas recevoir de notifications par courrier électronique, désactivez les notifications "Email". les notifications "Email".

Si vous préférez les notifications par courrier électronique, vous pouvez procéder de plusieurs façons. de gérer les choses.

Paramètres Repo

Chaque dépôt dispose d'un certain nombre de paramètres de notification :

  • Watching - you get notifications for everything. Any issue, any Pull Request, any comment. This is helpful if you really really super duper care about a repo
  • Not Watching - you will only get notifications if you're mentioned, or when you're participating (say, on your own Pull Request)
  • Ignoring - never ever ever be notified.

Vous pouvez vérifier les dépôts que vous surveillez sur l'ensemble de GitHub à partir de la page [page de surveillance] (https://github.com/watching).

Filtres de messagerie

Les emails de GitHub sont riches en informations qui peuvent vous aider à filtrer ce qui vous intéresse et ce qui ne vous intéresse pas. ce qui vous intéresse et ce qui ne vous intéresse pas.

An email's subject line includes the name of the repo in question.

  • Par exemple, [beeware/batavia] Add a new function serait une notification du repo Batavia. notification du repo Batavia

Si votre nom est cité :

  • le courriel sera envoyé en copie conforme à mention@noreply.github.com
  • le courriel contiendra la phrase suivante Vous recevez ce message parce que vous avez été mentionné.

Si vous n'êtes abonné qu'à un repo :

  • le courriel sera envoyé en copie à subscribed@noreply.github.com.

Ssshhhhhhhhhh

Si vous souhaitez mettre en sourdine un fil de discussion spécifique, vous pouvez cliquer sur le bouton "Unsubscribe" (se désabonner) sur n'importe quel problème ou demande d'extraction.

If you get email, you can click on 'Mute this thread'.
