إشعارات GitHub 101

إذن أنت الآن تستخدم GitHub، وهو يقوم بإعلامك. ماذا الآن؟

بمجرد أن تبدأ المساهمة في BeeWare، سيبدأ GitHub في بإخطارك بالأشياء.

هذا أمر جيد! ولكن يمكن أن يصبح الأمر مربكاً؛ لذا، إليك بعض النصائح النصائح لمساعدتك في محاربة الطوفان.

  • إذا كنت تفضل إشعارات البريد الإلكتروني فقط، قم بتعطيل إشعارات "الويب" "الويب".
  • إذا كنت تفضل عدم تلقي إشعارات البريد الإلكتروني، فقم بتعطيل إشعارات "البريد الإلكتروني" "البريد الإلكتروني".

إذا كنت تفضل إشعارات البريد الإلكتروني، فهناك مجموعة من الطرق التي يمكنك من خلالها التعامل مع الأمور.

إعدادات الريبو

يحتوي كل ريبو على عدد من إعدادات الإشعارات:

  • Watching - you get notifications for everything. Any issue, any Pull Request, any comment. This is helpful if you really really super duper care about a repo
  • Not Watching - you will only get notifications if you're mentioned, or when you're participating (say, on your own Pull Request)
  • Ignoring - never ever ever be notified.

يمكنك التحقق من الريبوسات التي تشاهدها عبر GitHub بأكملها من [صفحة المشاهدة] (https://github.com/watching).

مرشحات البريد الإلكتروني

رسائل البريد الإلكتروني ل GitHub غنية بالمعلومات التي يمكن أن تساعدك في تصفية ما ما تهتم به وما لا تهتم به.

An email's subject line includes the name of the repo in question.

  • على سبيل المثال: "إضافة دالة جديدة" سيكون إشعارًا من ريبو باتافيا إشعارًا من الريبو باتافيا

إذا تم ذكرك بالاسم

  • سيتم إرسال رسالة البريد الإلكتروني إلى "mention@noreply.github.com".
  • سيتضمن البريد الإلكتروني عبارة "أنت تتلقى هذا لأنه تم ذكرك".

إذا كنت مشتركاً في ريبو فقط:

  • سيتم إرسال البريد الإلكتروني إلى "subscribed@noreply.github.com".
