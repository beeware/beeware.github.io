Podium

基於標記的幻燈片簡報工具

開發人員會參加會議。參加會議時，他們需要幻燈片。

Unfortunately, while presentation tools like Keynote and PowerPoint are great for business presentations, they aren't well suited to the needs of developers. The mainstay of developer presentations -- code samples -- are generally painful to add to a Keynote presentation.

這些簡報工具也來自 WYSIWYG（所見即所得）傳統。 傳統。這可以很強大，因為它可以輕鬆地 將您想要的任何東西放到幻燈片上。但它也可能很痛苦，因為 您最終可能會花費所有時間將像素推到正確的位置、 而不是專注於演講內容。如果您想 更改樣式，您可能需要手動將更改套用到每一張 幻燈片。將內容與標記分離的教訓無法應用在 所見即所得的世界。

除此之外，Keynote 和 Powerpoint 的文件格式是 二進位 Blob -- 他們不適合版本控制、 協同編輯等等。

開發人員對此的回應是使用 HTML5。近年來 開發了許多基於 HTML 的簡報工具，例如 prezi、deck.js、 keydown，以及 showoff。這些工具利用 HTML5 的威力來製作全螢幕簡報。

然而，由於使用瀏覽器技術作為這些工具的基礎，這些工具 的一個非常重要的功能：簡報者模式。 模式。

Keynote 和 Powerpoint 的一大特色是它們不僅是 它們有簡報筆記和計時工具，而且 顯示給聽眾的內容與顯示給主持人的內容不同。 演示者的顯示不一樣。

許多這些工具也假設您有良好的 WiFi 連線、 並能夠在網路上即時顯示您的內容...... 如果您曾參加過開發者會議，您就會知道這是一個很冒險的 風險。
