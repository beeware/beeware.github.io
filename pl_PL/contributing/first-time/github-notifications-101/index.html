Powiadomienia GitHub 101

Korzystasz teraz z GitHub i otrzymujesz powiadomienia. Co teraz?

Gdy zaczniesz współtworzyć BeeWare, GitHub zacznie powiadamiać o różnych rzeczach.

To dobra rzecz! Ale może to być przytłaczające; więc oto kilka wskazówki, które pomogą ci walczyć z zalewem.

  • Jeśli wolisz tylko powiadomienia e-mail, wyłącz opcję "Web" powiadomienia.
  • Jeśli nie chcesz otrzymywać powiadomień e-mail, wyłącz opcję "E-mail". powiadomienia.

Jeśli wolisz powiadomienia e-mail, możesz to zrobić na kilka sposobów. sposób.

Ustawienia repo

Każde repozytorium ma wiele ustawień powiadomień:

  • Obserwowanie - otrzymujesz powiadomienia o wszystkim. Każda sprawa, każdy Pull Request, każdy komentarz. Jest to przydatne, jeśli naprawdę super duper zależy na repo
  • Not Watching - będziesz otrzymywać powiadomienia tylko wtedy, gdy zostaniesz wspomniany lub gdy bierzesz udział (powiedzmy, w swoim własnym Pull Request).
  • Ignorowanie - nigdy, przenigdy nie otrzymasz powiadomienia.

Możesz sprawdzić, jakie repozytoria obserwujesz na całym GitHubie, korzystając ze strony strona obserwowania.

Filtry e-mail

E-maile GitHub są bogate w informacje, które mogą pomóc w odfiltrowaniu tego, co na czym Ci zależy, a na czym nie.

Tematem wiadomości e-mail będzie dane repozytorium

  • Na przykład, [beeware/batavia] Add a new function będzie powiadomieniem z repo Batavia. powiadomienie z repozytorium Batavia

Jeśli jesteś wymieniony z imienia i nazwiska:

  • wiadomość e-mail zostanie wysłana jako cc'd na adres mention@noreply.github.com
  • e-mail będzie zawierał frazę "Otrzymujesz tę wiadomość, ponieważ zostałeś wspomniany".

Jeśli subskrybujesz tylko repozytorium:

  • wiadomość e-mail zostanie wysłana jako cc'd na adres subscribed@noreply.github.com.

Jeśli otrzymasz wiadomość e-mail, możesz kliknąć opcję "Wycisz ten wątek".
