Notifiche GitHub 101

Ora state usando GitHub e vi sta avvisando. Cosa succede ora?

Una volta che si inizia a contribuire a BeeWare, GitHub inizierà a a notificare le cose.

È una buona cosa! Ma può diventare opprimente. consigli per aiutarvi a combattere il diluvio.

  • Se si preferisce ricevere solo notifiche via e-mail, disabilitare le notifiche "Web". .
  • Se si preferisce non ricevere notifiche via e-mail, disabilitare le notifiche "Email". notifiche "Email".

Se si preferisce ricevere le notifiche via e-mail, ci sono diversi modi per gestire le cose.

Impostazioni Repo

Ogni repo ha una serie di impostazioni di notifica:

  • Watching - you get notifications for everything. Any issue, any Pull Request, any comment. This is helpful if you really really super duper care about a repo
  • Not Watching - you will only get notifications if you're mentioned, or when you're participating (say, on your own Pull Request)
  • Ignoring - never ever ever be notified.

È possibile controllare quali repository si stanno controllando in tutto GitHub dalla pagina [pagina di osservazione] (https://github.com/watching).

Email filters

Le e-mail di GitHub sono ricche di informazioni che possono aiutarvi a filtrare ciò che vi interessa e ciò che non vi interessa. di ciò che vi interessa e di ciò che non vi interessa.

An email's subject line includes the name of the repo in question.

  • Per esempio, [beeware/batavia] Aggiungi una nuova funzione sarebbe una notifica notifica dal repo Batavia

Se si viene citati per nome:

  • il messaggio di posta elettronica sarà inviato come cc a `mention@noreply.github.com
  • l'e-mail includerà la frase Ricevi questo messaggio perché sei stato menzionato".

Se si è iscritti solo a un repo:

  • l'e-mail sarà inviata a subscribed@noreply.github.com.
