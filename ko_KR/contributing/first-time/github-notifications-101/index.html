GitHub 알림 101

이제 GitHub를 사용 중이고 알림이 표시됩니다. 이제 어떻게 하나요?

BeeWare에 기여하기 시작하면 GitHub에서 다음과 같이 알림을 보내드립니다.

이것은 좋은 일입니다! 하지만 부담스러울 수 있으니, 여기 몇 가지 몇 가지 팁을 알려드리겠습니다.

  • 이메일 알림만 받으려면 "웹" 알림을 비활성화하세요. 알림을 비활성화합니다.
  • 이메일 알림을 받지 않으려면 "이메일" 알림을 비활성화하세요. 알림을 비활성화합니다.

이메일 알림을 선호하는 경우 다음과 같은 다양한 방법으로 처리할 수 있습니다. 처리할 수 있습니다.

리포지토리 설정

각 리포지토리에는 여러 가지 알림 설정이 있습니다:

  • 감시 중 - __모든 항목__에 대한 알림을 받습니다. 모든 이슈, 모든 풀 리퀘스트, 모든 댓글. 이 기능은 정말 정말 리포지토리에 매우 관심이 많을 때 유용합니다.
  • 보고 있지 않음 - 내가 언급된 경우에만 알림을 받거나 참여 중일 때(예: 직접 풀 리퀘스트에 참여 중일 때) 알림을 받습니다.
  • 무시 - 절대 알림을 받지 않습니다.

GitHub의 모든 리포지토리에서 어떤 리포지토리를 보고 있는지 확인하려면 감시 페이지에서 확인할 수 있습니다.

이메일 필터

GitHub 이메일에는 관심 있는 정보와 그렇지 않은 정보를 필터링하는 데 도움이 되는 관심 있는 항목과 그렇지 않은 항목을 필터링하는 데 도움이 되는 정보가 풍부합니다.

이메일 제목에는 해당 리포지토리의 제목이 포함됩니다.

  • 예를 들어, '[비웨어/바타비아] 새 기능 추가'는 리포지토리의 리포지토리의 알림이 될 것입니다.

이름이 언급된 경우

  • 이메일은 mention@noreply.github.com로 참조됩니다.
  • 이메일에는 다음과 같은 문구가 포함됩니다. '귀하가 언급되었으므로 이 이메일을 수신합니다'라는 문구가 포함됩니다.

리포지토리에만 가입한 경우:

  • 이메일은 subscribed@noreply.github.com로 참조됩니다.

쉿!

특정 스레드를 뮤트하려면 '구독 취소' 버튼을 클릭하면 됩니다.

이메일을 받은 경우 '이 스레드 뮤트하기'를 클릭하면 됩니다.

GitHub에는 알림에 도움이 되는 다양한 리소스가 있습니다.
