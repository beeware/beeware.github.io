VOC

파이썬 바이트코드를 자바 바이트코드로 변환하는 트랜스파일러입니다.

VOC는 크게 두 가지 기능을 합니다:

  • Python 3.4 이상 소스 파일을 Java 클래스 파일로 컴파일하여 다음을 수행할 수 있습니다. JVM(Android의 VM 포함)에서 Python 코드를 실행할 수 있습니다.
  • 프로그래밍 방식으로 Java 클래스 파일을 생성할 수 있는 API를 제공합니다.

아직 파이썬 3.4를 완전히 준수하는 구현은 아닙니다. 일부 언어 기능(예: '비로컬' 및 일부 내장 함수)이 아직 구현되어야 합니다. 구현해야 하는 일부 언어 기능이 있으며, 표준 라이브러리 구현도 표준 라이브러리 구현만 있습니다. 하지만 간단한 Python 프로그램을 간단한 파이썬 프로그램을 변환하고, 심지어 간단한 안드로이드 애플리케이션.

So, why is it called "VOC"?

VOC는 "흔적 출력 컴파일러"의 줄임말입니다. 또는 "Vexing Obtuse 컴파일러". 또는 "가치 있는 모호한 컴파일러"라고도 합니다. 또는 "다양한 오셀롯 캘리브레이터". 선택은 여러분의 몫입니다.

역사적으로 VOC는 Vereenigde Oostindische Compagnie (VOC) 또는 네덜란드어 동인도 회사로, 종종 세계 최초의 다국적 기업입니다. 또한 최초로 주식을 발행한 회사이기도 합니다. 주식을 발행하고 그 주식의 거래를 촉진한 최초의 회사이기도 합니다. 이 회사는 21 아시아, 주로 네덜란드령 동인도 제도인 스파이스 제도에서 무역 활동을 수행할 수 있는 21년 독점권을 부여받았습니다. 네덜란드 동인도 제도. 그들은 주요 무역항을 설립했습니다. 자바 섬(현재 인도네시아의 일부인 인도네시아).

VOC는 주주들에게 매우 수익성이 높은 사업이었지만, 식민지배를 위해 식민지 세력으로 운영되었습니다. 현대 역사가들은 그들의 사업에 대한 비판 관행](https://en.wikipedia.org/wiki/Dutch_East_India_Company#Criticism), 여기에는 원주민 착취(노예 노동력 사용 포함), 노예 착취( 노예 노동), 노예 무역, 폭력 사용, 환경 파괴(삼림 파괴 포함) 파괴(삼림 벌채 포함).

유럽인들이 호주를 발견하는 데도 VOC가 중요한 역할을 했으며, 서호주 연안에는 서호주 해안에는 많은 VOC 난파선이 있습니다.

프로젝트 창립자가 서호주 출신이기 때문에 프로젝트에서 VOC라는 이름을 선택했습니다. 서호주 출신이며, VOC는 서호주의 역사에서 식민지배가 아닌 그 주의 역사에서 강한 존재감을 가지고 있기 때문입니다. 또한 VOC가 엔터프라이즈 Java 회사였기 때문에 편리한 말장난이기도 했습니다. 엔터프라이즈 Java 회사였기 때문입니다.

이는 회사의 착취적인 비즈니스 관행에 대한 회사의 착취적인 비즈니스 관행을 지지하거나 미화하는 것으로 간주되어서는 안 됩니다.
