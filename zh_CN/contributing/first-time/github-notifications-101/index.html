GitHub 通知 101

您现在正在使用 GitHub，但它正在通知您。现在怎么办？

一旦您开始为 BeeWare 做出贡献，GitHub 就会开始通知您相关事宜。

这是一件好事！但它也会让人应接不暇；因此，这里有一些小贴士，帮助您与这些洪水作斗争。

  • 如果您只喜欢收到电子邮件通知，请禁用“Web”通知。
  • 如果不想收到电子邮件通知，请禁用“Email”通知。

如果您确实更喜欢电子邮件通知，您有很多处理方式。

仓库设置

每个仓库都有许多通知设置：

  • Watching - 你会收到关于所有事情的通知。任何问题、任何拉取请求、任何评论。如果你真的真的真的特别关心某个仓库，这才有用。
  • Not Watching - 只有当你被提及，或当您参与时（例如，在您自己的拉取请求中）才会收到通知。
  • Ignored - 永远不会收到通知。

您可以从关注页面查看您正在关注的所有 GitHub 仓库。

电子邮件过滤器

GitHub 电子邮件包含丰富的信息，可以帮助您筛选出您关心的内容或不关心的内容。

电子邮件的主题将包括有关仓库的名称。

  • 例如，[beeware/batavia] Add a new function 将是从 Batavia 仓库来的通知

如果有人提到你的名字：

  • 电子邮件将抄送 mention@noreply.github.com
  • 电子邮件将包括 You are receiving this because you were mentioned. 语句

如果您只是订阅了一个仓库的通知：

  • 电子邮件将被抄送至 subscribed@noreply.github.com

嘘

如果您想将特定帖子静音，可以点击任何问题或拉取请求上的“Unsubscribe”按钮。

如果你收到电子邮件，你可以点击“Mute this thread”。

GitHub 上有大量资源可帮助您处理通知。
