Imposter Syndrome

Sie glauben nicht, dass Sie bereit sind, einen Beitrag zu Open Source zu leisten? Sie irren sich.

Vielleicht gibt es eine kleine Stimme in Ihrem Kopf, die Ihnen sagt, dass dass Sie noch nicht so weit sind, dass Sie noch ein Tutorial machen müssen, dass Sie noch nicht Open-Source-Mitarbeiter zu sein. Schließlich bist du ja nur ein Anfängerin. Was könntest du einem Projekt wie BeeWare schon bieten?

Ich versichere Ihnen - die kleine Stimme in Ihrem Kopf hat Unrecht. Wenn Sie Code schreiben können, können Sie Code zu Open Source und zu BeeWare beitragen.

Sie sind auch nicht die erste Person, die diese Gedanken hat. Sogar die Mitglieder des Kernteams dieses Projekts haben diese Gedanken von Zeit zu Zeit von Zeit zu Zeit. Man nennt das "Impostersyndrom", und es ist ein sehr häufiges Problem. Die gute Nachricht ist, dass wir Ihnen helfen können, es zu überwinden.

Diese Anleitung dient dazu, dass Sie genau wissen, welchen Prozess Sie befolgen müssen, damit Ihr Patch zusammengeführt wird. Zusätzlich zu diesen Verfahrensanweisungen hat dieses Projekt auch einen Verhaltenskodex. Dieser Verhaltenskodex Verhaltenskodex soll Ihnen die Sicherheit geben, dass Sie, egal welche Fehler Sie machen, Sie werden mit Respekt behandelt. Jeder macht Fehler - das ist ein natürlicher Teil des Lernens. Unser Versprechen an Sie ist, dass wir hier sind um Ihnen beim Lernen zu helfen, und nicht, um Sie zu beleidigen oder herabzusetzen, weil Sie lernen.

Zu Open Source beizutragen bedeutet auch nicht nur, Code zu schreiben. Sie können mithelfen, indem Sie Dokumentationen und Tests schreiben, oder sogar Feedback über das Projekt geben (und ja - das schließt auch Feedback über den Beitragsprozess). Einige dieser Beiträge sind vielleicht die für das Projekt als Ganzes am wertvollsten sein, weil Sie das Projekt mit mit frischen Augen an das Projekt heran, so dass Sie die Fehler und Annahmen sehen können, die die erfahrene Mitwirkende übersehen haben.

Sie können auch keinen Schaden anrichten - weder an Ihrem eigenen Computer noch an dem Projekt als Ganzes. BeeWare-Projekte berühren keinen Teil Ihres Computers oder des Betriebssystems an, die ernsthaften Schaden anrichten könnten. Im schlimmsten Schlimmstenfalls haben Sie ein paar zusätzliche Dateien auf Ihrer Festplatte, die Sie anschließend einfach löschen können. Und jeder Beitrag, den Sie Beitrag, den Sie bei BeeWare einreichen, wird geprüft, bevor er in das "offizielle" Projekt integriert wird. "offiziellen" Projekt integriert wird, und Sie erhalten eine Rückmeldung, die Ihnen hilft, eventuelle Probleme zu beheben, die möglicherweise bestehen.

Scheuen Sie sich also nicht, einen Beitrag zu leisten. Wenn Sie bis hierher gekommen sind, haben Sie haben Sie bewiesen, dass Sie ein Interesse daran haben, etwas beizutragen - und das ist alles, was Sie brauchen. Wir können Ihnen den Rest des Weges helfen.

Jetzt ist es an der Zeit, die Ärmel hochzukrempeln und ein Projekt auszuwählen, zu dem Sie beitragen.
