GitHub Notifications 101

Sie verwenden jetzt also GitHub und werden benachrichtigt. Was nun?

Sobald Sie anfangen, zu BeeWare beizutragen, wird GitHub beginnen Sie über Dinge zu benachrichtigen.

Das ist eine gute Sache! Aber es kann auch überwältigend werden; deshalb hier einige Tipps, die Ihnen helfen, die Flut zu bewältigen.

  • Wenn Sie nur E-Mail-Benachrichtigungen bevorzugen, deaktivieren Sie die "Web"-Benachrichtigungen Benachrichtigungen.
  • Wenn Sie keine E-Mail-Benachrichtigungen erhalten möchten, deaktivieren Sie die Option "E-Mail". Benachrichtigungen.

Wenn Sie E-Mail-Benachrichtigungen bevorzugen, gibt es eine Reihe von Möglichkeiten, wie Sie handhaben.

Repo-Einstellungen

Jedes Repo hat eine Reihe von Benachrichtigungseinstellungen:

  • Beobachten - Sie erhalten Benachrichtigungen für __Alles__. Jedes Problem, jeden Pull Request, jeden Kommentar. Das ist hilfreich, wenn du dich wirklich sehr super duper an einem Repo interessiert bist
  • Not Watching - du bekommst nur Benachrichtigungen, wenn du erwähnt wirst, oder wenn Sie beteiligt sind (z.B. an Ihrem eigenen Pull Request)
  • Ignorieren - Sie werden nie und nimmer benachrichtigt.

Sie können überprüfen, welche Repos Sie in ganz GitHub beobachten, indem Sie die [Beobachtungsseite] (https://github.com/watching).

E-Mail-Filter

GitHub-E-Mails sind reich an Informationen, die Ihnen helfen können, herauszufiltern, was was Sie interessiert und was nicht.

Die E-Mails tragen die Betreffzeile des betreffenden Repos

  • Zum Beispiel wäre [beeware/batavia] Add a new function eine Benachrichtigung aus dem Repo Batavia

Wenn Sie namentlich erwähnt werden:

  • die E-Mail wird mit cc an "mention@noreply.github.com" gesendet
  • die E-Mail enthält die Formulierung "Sie erhalten diese E-Mail, weil Sie erwähnt wurden".

Wenn Sie nur ein Repo abonniert haben:

  • die E-Mail wird an "subscribed@noreply.github.com" weitergeleitet

Ssshhhhhhhhh

Wenn Sie einen bestimmten Thread stummschalten möchten, können Sie auf die Schaltfläche 'Abmelden' in jeder Ausgabe oder jedem Pull Request klicken.

Wenn Sie eine E-Mail erhalten, können Sie auf die Schaltfläche "Dieses Thema stummschalten" klicken.
