Beitragender

Möchten Sie zu BeeWare beitragen? Jeder ist willkommen, mitzuhelfen!

Es gibt viele Möglichkeiten, mit BeeWare zu helfen.

Erstmalig beitragende Personen

Wenn Sie neu im Projekt sind (oder sogar ganz neu bei Open Source im Allgemeinen im Allgemeinen), beginnen Sie am besten hier. Jeder kann zu Open Source beitragen, und wir zeigen Ihnen, wie.

Code Beiträge

Ihre Code-Beiträge sind willkommen! Bitte machen Sie sich vertraut mit unserem Entwicklungsprozess.

Wenn Sie nach etwas suchen, an dem Sie arbeiten können, und nicht wissen, wo Ihre Arbeit willkommen wäre, versuchen Sie es mit einem Blick auf den Ticket-Tracker für ein Projekt von Interesse. Normalerweise gibt es ein paar Tickets mit bekannten Problemen; jedes Ticket ist ein Kandidat für eine Lösung.

Wenn Sie zum ersten Mal einen Beitrag leisten, sind einige Tickets auch mit folgenden Tags versehen [[gute erste issue]] (https://github.com/search?q=user%3Abeeware+label%3A%22good%20first%20issue%22+is%3Aissue+is%3Aopen&type=issues). Es handelt sich um spezielle Ausgaben, die ausgewählt wurden, weil sie relativ einfache Einführungen in das Projekt sind, und das BeeWare-Team unterstützt jeden, der zum ersten Mal einen Patch für eines dieser Themen einreicht diese Themen.

Verwendung der Plattform

Verwenden Sie Windows oder verschiedene Linux-Varianten? Sind Sie in der Lage ein Projekt oder eine Anwendung auf Ihrem System installieren? Sind Sie auf irgendwelche Probleme gestoßen?

Wenn ja, aktualisieren Sie bitte die Dokumentation, um zu zeigen, wie Sie es zum Funktionieren gebracht haben. oder melden Sie ein Problem, wenn Sie einen Fehler gefunden haben, den Sie nicht beheben können.

Dokumentation

Ist die Dokumentation auf dem neuesten Stand? Sind Sie der Meinung, dass Dinge anders formuliert werden könnten anders formuliert werden? Gibt es fehlende Abschnitte? Haben Sie eine Idee für ein Tutorial, das geschrieben werden könnte? Bitte reichen Sie einen Pull Request ein!

Ist etwas falsch oder fehlt etwas auf dieser Website? Bitte zögern Sie nicht Änderungen vorzunehmen und einen Pull-Request ein!

Helfen Sie bei der Übersetzung und Aktualisierung dieser Website

Sprechen Sie eine andere Sprache als Englisch und möchten Sie anderen helfen anderen einen besseren Zugang zur BeeWare-Dokumentation zu ermöglichen? Besuchen Sie die Abschnitt Übersetzungen, um zu erfahren, wie Sie Übersetzungen der BeeWare-Dokumentation beitragen können.

Erstellen Sie eine echte Anwendung!

Letztendlich ist BeeWare eine Reihe von Werkzeugen zur Erstellung von Anwendungen. Eine der beste Weg, unsere Fortschritte zu bewerten, ist, zu sehen, wie Menschen mit den Anwendungen mit den BeeWare-Tools erstellen. Wenn Sie also eine Idee für eine Anwendung haben Anwendung haben, die Sie schon immer haben wollten oder brauchten - versuchen Sie, sie zu entwickeln!

Bei der Erstellung dieser Anwendung werden Sie wahrscheinlich Fehler finden, fehlende Funktionen oder Reibungen im Entwicklungsprozess. Diese Probleme können dann in Fehlerberichte oder Patches umgewandelt werden.

Etwas anderes

Wenn Sie eine eigene Idee für etwas haben, oder wenn Sie keine Tickets sehen, oder wenn Sie einen Mentor für ein Ticket suchen, oder wenn Sie nicht sicher sind, wie vorgehen sollen, kontaktieren Sie die Projektbetreuer unter Mastodon oder Discord. Sie können uns auch finden sprinting bei verschiedenen Veranstaltungen, bei denen sich Erstbeitragsleistende eine challenge Münze verdienen können. Die Projektbetreuer lieben Zusammenarbeit, besonders mit neuen Mitwirkenden, und beantworten gerne Fragen oder helfen Ihnen bei Problemen, die Sie möglicherweise haben.

Alle Beiträge willkommen

Es geht nicht nur um den Code. Ein erfolgreiches Softwareprojekt erfordert Dokumentation, Designkenntnisse, Feedback und Fehlerberichte. Die BeeWare Gemeinschaft erkennt an, dass alle Beiträge wichtig sind - nicht nur die als Pull-Request auf GitHub eingehen.

Sprints

Schließen Sie sich dem BeeWare-Projekt in einem Sprint an? Ein Sprint ist ein unstrukturierte Zusammenkunft, bei der wir an BeeWare und verwandten Projekten arbeiten, und neue Mitwirkende in den Beitragsprozess einführen. Wenn dies Sie zum ersten Mal an einem Sprint teilnehmen, gibt es bei BeeWare einen Leitfaden für die ersten Schritte mit BeeWare-Sprints.
